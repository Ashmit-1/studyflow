# Contributing to StudyFlow

Thanks for your interest in contributing to **StudyFlow** 🚀
StudyFlow is a full-stack study planning app built with **React**, **FastAPI**, and **MySQL**.

Beginner contributors are welcome.

---

## 🛠️ Tech Stack

* Frontend: React + Vite
* Backend: FastAPI
* Database: MySQL
* Auth: JWT + bcrypt
* Styling: CSS / Tailwind CSS

---

## 🚀 How to Contribute

### 1. Fork the Repository

Click the **Fork** button on GitHub.

---

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/studyflow.git
cd studyflow
```

---

### 3. Create a New Branch

```bash
git checkout -b feature/your-feature-name
```

Example:

```bash
git checkout -b feature/navbar-improvement
```

---

### 4. Make Your Changes

You can contribute by:

* fixing bugs
* improving UI
* updating documentation
* adding components
* improving CSS
* creating new features
* testing existing features

---

### 5. Check Git Status

```bash
git status
```

---

### 6. Add and Commit Changes

```bash
git add .
git commit -m "feat: add your feature description"
```

Use clear commit messages.

Examples:

```bash
feat: add responsive navbar
fix: resolve login toast issue
docs: update setup instructions
style: improve dashboard layout
refactor: clean student dashboard component
```

---

### 7. Push Your Branch

```bash
git push origin feature/your-feature-name
```

---

### 8. Create a Pull Request

Go to your fork on GitHub and click **Compare & Pull Request**.

In your PR, explain:

* what you changed
* why you changed it
* screenshots if UI changed

---

## ✅ Good First Issues

Beginner-friendly contribution ideas:

* improve README
* fix typos
* improve navbar styling
* add loading spinner
* improve toast notifications
* add dashboard cards
* improve responsive design
* add comments to complex code
* improve form validation messages

---

## 📦 Project Setup

### Backend

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app:app --reload
```

For macOS/Linux:

```bash
source .venv/bin/activate
```

---

### Frontend

```bash
cd FrontEnd
npm install
npm run dev
```

---

## 🧪 Before Submitting

Please check:

* app runs without errors
* no unnecessary console logs
* code is readable
* commit message is clear
* screenshots added for UI changes

---

## 🌿 Branch Naming

Use simple branch names:

```bash
feature/navbar-ui
fix/login-error
docs/readme-update
style/dashboard-css
```

---

## 📝 Commit Message Style

Use conventional commits:

```bash
feat: add new feature
fix: fix bug
docs: update documentation
style: update styling
refactor: improve code structure
chore: update config or dependencies
```

---

## 🤝 Code of Conduct

Be respectful and helpful.

Everyone starts somewhere.
Beginners are welcome here.
